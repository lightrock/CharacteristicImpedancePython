# Friston, Free Energy, and the Closure Machine

## Working thesis

Friston should not be treated as an ordinary BBQ target.

He is close to the right machine-level instinct:

```text
systems persist by maintaining a boundary
boundaries imply inside/outside accounting
perception and action are coupled
living systems act to keep themselves within viable states
```

That is useful. It is not just neuroscience decoration. The Free Energy Principle (FEP), active inference, and Markov blankets are serious attempts to describe how systems remain coherent in a changing world.

But the process-first critique is:

> Friston explains how systems stay real once there is a system. He does not yet explain why reality is made of closure-capable systems in the first place.

---

## 1. The win

Friston's strongest move is persistence through boundary.

The basic pattern is:

```text
internal states
sensory states
active states
external states
```

A Markov blanket separates internal and external states through sensory and active boundary states. That is powerful because it formalizes the difference between a system and its environment without requiring the system to be a little object sitting in naive space.

Active inference then adds the action/perception loop:

```text
sample world
update internal model
act to reduce expected surprise / free energy
maintain viable state
repeat
```

So the win is real:

```text
boundary matters
persistence matters
self-maintenance matters
perception and action are one loop
systems are not passive observers of a ready-made world
```

This is close to the closure-machine language.

---

## 2. The gap

The gap is that free-energy minimization is usually framed after a system already exists.

That leaves the harder substrate questions open:

```text
What makes a boundary admissible?
What makes a system persist as a system?
What selects the degrees of freedom that count as internal/external?
What makes the Markov blanket physically real rather than just a useful partition?
What is the running process underneath surprise minimization?
What produces the clock/tick/order in which active inference occurs?
```

From the process-first view, Friston starts one layer too late.

He gives a powerful account of maintenance:

```text
system -> boundary -> inference/action -> persistence
```

But the closure-machine wants the prior account:

```text
running process -> admissible boundary -> closure -> persistence -> system -> inference/action
```

That reversal matters.

---

## 3. The process-first repair

The repair is not to reject Friston.

The repair is to lower his machinery onto a physics-first closure substrate.

```text
running process
-> relation update
-> boundary candidate
-> closure/admissibility test
-> persistent system
-> Markov blanket
-> active inference
-> rendered observer geometry/time
```

Under this repair, a Markov blanket is not merely a statistical partition. It is a closure receipt.

A system is not just something that minimizes free energy. It is something that has already passed enough closure tests to persist as a system capable of minimization.

The compact repair:

> Free energy explains how a system keeps its shape. Closure explains why there is a system with a shape to keep.

---

## 4. Relation to Fundamental Pi

[`fundamentalPi.md`](fundamentalPi.md) asks for the return/closure operation underneath phase, rotation, and geometric completion.

Friston's persistence machinery can be read as a high-level biological/cognitive example of the same deeper requirement:

```text
state
-> transformation
-> comparison
-> correction/action
-> maintained boundary
-> persistence
```

But Fundamental Pi pushes earlier:

```text
state
-> transformation accumulator
-> return comparison
-> closure class
```

So the Friston repair is:

```text
FEP/active inference = maintenance dynamics after system closure
Fundamental Pi = primitive return/closure test before system closure
```

That gives a clean stack:

```text
pre-logical process
-> closure class
-> persistent boundary
-> Markov blanket
-> active inference
-> biological/cognitive agency
```

## 5. PFEM audit

PFEM separates the evidence species.

Friston's framework has strong formal and modeling value, but PFEM should not allow the model to silently become substrate ontology.

```text
variational formalism != physical substrate
surprise minimization != process drive
Markov blanket partition != admissibility proof
biological persistence != fundamental closure rule
optimization language != machine receipt
```

The PFEM demand:

> Show the receipt that a boundary is physically admissible, not just statistically useful.

---

## 6. The BBQ line

Friston found a real thing:

```text
systems persist by maintaining boundaries through action/perception loops
```

The BBQ is not that he is wrong.

The BBQ is that this is still downstream of the substrate machine.

```text
Friston gives persistence after the system exists.
Process-first closure asks why the system can exist at all.
```

Compact line:

> Friston found how systems stay real. Closure explains why there are systems that can stay real.

Friendly nuke:

```text
Free energy is not the engine.
It is the thermostat reading of a system already holding together.
The deeper machine is the closure process that makes a system possible before it can minimize anything.
```
